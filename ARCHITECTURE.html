Data Flow

1
User Interaction (Frontend)

User selects filters (warehouse, category, status, time period) or navigates to a view in the Vue 3 application

2
Composable Updates

useFilters() composable updates reactive refs with selected filter values and triggers watchers

3
API Call

Axios client in api.js constructs query parameters and sends HTTP request to FastAPI endpoint

4
Backend Processing

FastAPI endpoint validates input, applies filters to in-memory data using apply_filters() utility

5
Pydantic Validation

Response data is validated against Pydantic models (InventoryItem, Order, etc.) for type safety

6
Response Delivery

JSON response sent back to frontend with appropriate HTTP status code

7
State Update

Axios response triggers component state update in loading/error/data refs

8
UI Rendering

Vue's reactivity system re-renders components with new data, computed properties recalculate

Data Models

Core Models

InventoryItem

Fields: id, sku, name, category, warehouse, quantity_on_hand, reorder_point, unit_cost, location, last_updated

Purpose: Represents individual products in stock across warehouses

Order

Fields: id, order_number, customer, items, status, order_date, expected_delivery, total_value, warehouse, category

Purpose: Customer orders with fulfillment status tracking

DemandForecast

Fields: id, item_sku, item_name, current_demand, forecasted_demand, trend, period

Purpose: Predictive analytics for inventory planning

BacklogItem

Fields: id, order_id, item_sku, item_name, quantity_needed, quantity_available, days_delayed, priority, has_purchase_order

Purpose: Unfulfilled orders requiring attention

PurchaseOrder

Fields: id, backlog_item_id, supplier_name, quantity, unit_cost, expected_delivery_date, status, created_date

Purpose: Restocking orders from suppliers

Transaction

Fields: id, date, vendor, category, amount, order_id, status

Purpose: Financial transaction history for audit trail

Development Best Practices

Frontend

  • ✅ Use Composition API for all components
  • ✅ Unique keys in v-for (never index)
  • ✅ Validate dates before parsing
  • ✅ Keep components under 200 lines
  • ✅ Use computed properties for derived data
  • ✅ Centralize API calls in api.js
  • ✅ Always handle loading and error states

Backend

  • ✅ Validate all input with Pydantic models
  • ✅ Filter on copies, never mutate original data
  • ✅ Use consistent filter parameter names
  • ✅ Check for 'all' value to skip filter
  • ✅ Return proper HTTP status codes
  • ✅ Include error details in responses
  • ✅ Write tests for new endpoints

Data Management

  • ✅ Keep JSON files well-formatted
  • ✅ Use consistent date formats (YYYY-MM-DD)
  • ✅ Update Pydantic models when JSON changes
  • ✅ Validate data consistency across files
  • ✅ Ensure SKU references are valid
